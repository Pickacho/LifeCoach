# API Documentation Standard

1. **Inputs/Outputs**: Explicitly list all route parameters, query strings, and body payload structures.
2. **Status Codes**: Define expected success (200, 201) and failure (400, 401, 404, 500) codes with example response bodies.
3. **Format**: Use Markdown code blocks with JSON structures for all payload examples.
